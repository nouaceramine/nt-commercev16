# جسر NT Commerce — تشغيل الشحن عبر مودم GSM

برنامج بسيط يعمل على حاسوب المحل يتصل بمودم GSM USB ويُنفّذ عمليات الشحن تلقائياً.

---

## طريقتان للتثبيت

| | **الطريقة أ — ملف .exe (للمبتدئين ✅ موصى بها)** | **الطريقة ب — Python (للمطورين)** |
|---|---|---|
| يتطلب | لا شيء — فقط Windows 10/11 | Python 3.10+ |
| الإعداد | نقرة مزدوجة + معالج رسومي | تعديل config.json يدوياً |
| التشغيل | من علبة النظام (System Tray) | من سطر الأوامر |

---

## الطريقة أ — ملف .exe (نقرة مزدوجة)

### ما تحتاجه
- حاسوب Windows 10 أو 11
- مودم GSM USB (Huawei E153، E3131، ZTE، Sierra…)
- شريحة جملة لكل شبكة (موبيليس / جازي / أوريدو)
- اتصال بالإنترنت

### خطوات التثبيت

**الخطوة 1 — تركيب المودم**
1. أدخل شريحة الجملة في المودم
2. وصّل المودم بمنفذ USB
3. افتح **مدير الأجهزة** (ابحث عنه في قائمة ابدأ)
4. ابحث عن **منافذ COM** — لاحظ الرقم مثل **COM3** أو **COM4**

**الخطوة 2 — تشغيل برنامج الجسر**
1. انقر نقرة مزدوجة على **NT_Commerce_Bridge.exe**
2. سيفتح معالج الإعداد تلقائياً

![معالج الإعداد](docs/wizard_screenshot.png)

**الخطوة 3 — أدخل البيانات في المعالج**
1. **رابط التطبيق** — مثال: `https://my-shop.replit.app`
2. **Tenant ID** — من لوحة الإدارة ← إعدادات الشحن
3. **المفتاح السري** — من نفس الصفحة ← مفتاح الجسر
4. **منفذ COM** لكل مودم (اتركه فارغاً إن لم تكن لديك شريحة لتلك الشبكة)
5. اضغط "**اختبار الاتصال**" للتأكد
6. اضغط "**حفظ وبدء التشغيل**"

**الخطوة 4 — البرنامج يعمل في الخلفية**

ستظهر أيقونة في **علبة النظام** (System Tray — أسفل يمين الشاشة):
- 🟢 **أخضر** — متصل بالسحابة ويعمل
- 🔴 **أحمر** — منقطع عن الإنترنت أو السحابة
- 🟡 **أصفر** — جاري الاتصال

انقر بالزر الأيمن على الأيقونة للوصول إلى:
- **عرض السجل** — لمتابعة العمليات
- **إعادة الإعداد** — لتغيير بيانات الاتصال
- **إنهاء** — لإيقاف البرنامج

### تشغيل البرنامج مع بدء Windows تلقائياً

1. اضغط `Win + R` ← اكتب `shell:startup` ← اضغط Enter
2. انسخ ملف **NT_Commerce_Bridge.exe** إلى المجلد الذي فُتح

## الطريقة ب — تشغيل من Python (للمطورين)

### المتطلبات

| المتطلب | التفاصيل |
|---|---|
| Python 3.10+ | [تحميل Python](https://www.python.org/downloads/) |
| مودم GSM USB | أي مودم يدعم أوامر AT |
| شرائح الجملة | شريحة واحدة لكل شبكة |
| اتصال بالإنترنت | للتواصل مع التطبيق السحابي |

### الخطوة 1 — تركيب المودم والشريحة

1. أدخل شريحة الجملة في المودم
2. وصّل المودم بمنفذ USB بالحاسوب
3. تأكد أن Windows يتعرف عليه (ابحث عن "مدير الأجهزة" → منافذ COM)
4. لاحظ رقم المنفذ مثل **COM3** أو **COM4** — ستحتاجه في الإعداد

> **تلميح Linux:** المنفذ سيكون `/dev/ttyUSB0` أو `/dev/ttyUSB1`

---

### الخطوة 2 — تثبيت Python والمكتبات

```bash
pip install pyserial requests
```

---

### الخطوة 3 — الحصول على المفتاح السري

1. افتح لوحة إدارة المحل في المتصفح
2. اذهب إلى **الإعدادات ← إعدادات الشحن ← مفتاح الجسر**
3. انسخ المفتاح (أو أنشئ واحداً جديداً إن لم يوجد)
4. احتفظ أيضاً بـ **معرّف المستأجر (Tenant ID)** من نفس الصفحة

---

### الخطوة 4 — إعداد config.json

افتح ملف `bridge/config.json` بأي محرر نصوص وعدّل القيم:

```json
{
  "cloud_url":     "https://اسم-المتجر.replit.app",
  "tenant_id":     "معرّف-المستأجر",
  "bridge_secret": "المفتاح-السري",

  "modems": [
    {
      "network":      "mobilis",
      "port":         "COM3",
      "pin":          "",
      "balance_ussd": "*100#",
      "slot_id":      1
    },
    {
      "network":      "djezzy",
      "port":         "COM4",
      "pin":          "",
      "balance_ussd": "*555#",
      "slot_id":      2
    }
  ]
}
```

**ملاحظات على الحقول:**

| الحقل | الوصف |
|---|---|
| `network` | اسم الشبكة: `mobilis` أو `djezzy` أو `ooredoo` |
| `port` | منفذ COM للمودم (COM3، COM4…) أو `/dev/ttyUSB0` على Linux |
| `pin` | رمز PIN للشريحة — اتركه فارغاً `""` إن لم يوجد قفل |
| `balance_ussd` | كود USSD للاستعلام عن الرصيد |
| `slot_id` | رقم الشريحة في التطبيق (1=موبيليس، 2=جازي، 3=أوريدو) |

> إن لم تكن لديك شريحة لشبكة معينة، احذف كتلتها من قائمة `modems`.

### تشغيل البرنامج عند بدء الحاسوب (Windows)

1. أنشئ ملف `start_bridge.bat` بجانب `bridge.py`:
   ```batch
   @echo off
   cd /d %~dp0
   python bridge.py >> bridge.log 2>&1
   ```
2. اضغط `Win + R` ← اكتب `shell:startup` ← اضغط Enter
3. ضع اختصاراً لـ `start_bridge.bat` في المجلد الذي فُتح

---

### تشغيل البرنامج عند بدء الجهاز (Linux)

```bash
crontab -e
# أضف السطر التالي:
@reboot cd /path/to/bridge && python bridge.py >> bridge.log 2>&1 &
```

## بناء ملف .exe (للمطورين فقط)

لبناء ملف `NT_Commerce_Bridge.exe` من المصدر على Windows:

```batch
cd bridge
build.bat
```

أو يدوياً:

```batch
pip install pyinstaller pystray Pillow pyserial requests
pyinstaller bridge.spec
```

الناتج يكون في `dist\NT_Commerce_Bridge.exe`.

## استكشاف الأخطاء

### ❌ "المودم لا يستجيب على AT"
- تحقق من رقم COM في مدير الأجهزة
- أغلق أي برنامج آخر يستخدم نفس المنفذ (مثل HyperTerminal)
- جرب فصل المودم وإعادة تركيبه

### ❌ "رُفض الوصول (403)"
- تحقق من `bridge_secret` في config.json — يجب أن يطابق ما في إعدادات التطبيق
- تحقق من `tenant_id`

### ❌ "لا يمكن الاتصال بالسحابة"
- تأكد من اتصال الإنترنت
- تأكد من صحة `cloud_url` (بدون `/` في النهاية)

### ❌ "الشريحة محمية بـ PIN"
- أدخل رمز PIN في حقل PIN بمعالج الإعداد أو في ملف config.json

### ⚠ الشحن يُنفَّذ لكن يُعلَّم "فشل"
- تحقق من ملف `bridge.log` لرؤية الرد الفعلي من المودم

### ⚠ أيقونة علبة النظام لا تظهر
- تأكد أنك لا تُشغّل البرنامج من على سطح المكتب مباشرة — انقر بالزر الأيمن واختر "تشغيل كمسؤول"
- أو راجع `bridge.log` لرؤية أي خطأ
